meanings

  • commited state: it iis a command used to save your changes to the local repository

  • modified state: a file in git repository has been changed and not yet staged for commit

  • staged state: it is the process of preparing changes for commit

  • brancing: it allows you to develop features, fix bugs or safely experiment with new ideas in a containd area of repository

  • merging: combining changes from one branch into another branch

  • pull request: it is a proposal to merge a set of changes from one branch into another

  • collaboration: it is the way different people can work together on the same project together

  • cloning: it downloads a repository to your local machine for local development
